## ✅ Step 2：AAB 生成（PWA Builder ・ 30 分）

### 2.1 PWA Builder にアクセス
1. `https://www.pwabuilder.com/` を開く
2. URL 欄に `https://rainybrainch.github.io/pomojikan/` を入力
3. 「Start」をクリック

### 2.2 PWA スコア確認
- Manifest, Service Worker, Security の 3 つのスコアが表示される
- 全て緑（または不足項目があれば修正）

### 2.3 Android パッケージ生成
1. 上部の「Package for stores」をクリック
2. 「Android」タブを選択
3. 設定：
   - **Package ID**: `io.github.rainybrainch.pomojikan`
   - **App name**: `ぽもじかん`
   - **Launcher name**: `ぽもじかん`
   - **App version**: `1.0.0`
   - **App version code**: `1`
   - **Display mode**: `Standalone`
   - **Status bar color**: `#0d1f2e`
   - **Background color**: `#07111c`
4. 「Generate」をクリック
5. ZIP ファイルをダウンロード（中身に AAB ＋ signing.keystore ＋ instructions）

### 2.4 ⚠ 超重要：keystore を絶対紛失しない
- `signing.keystore` ファイルと PASSWORD は **絶対に紛失しない**
- 紛失すると **同じアプリの更新ができなくなる**（新規アプリとして再申請が必要）
- 推奨：Google Drive ＋ ローカル両方にバックアップ

## ✅ Step 3：Play Console 設定（30 分）

### 3.1 アプリ作成
1. `https://play.google.com/console/` を開く
2. 「アプリを作成」をクリック
3. アプリ情報を入力：
   - **アプリ名**: `ぽもじかん`
   - **デフォルトの言語**: 日本語
   - **アプリ or ゲーム**: アプリ（ゲーム要素はあるが、生産性カテゴリ）
   - **無料 or 有料**: 無料
   - **宣言**: すべて該当する項目にチェック

### 3.2 ストア掲載情報
- **短い説明**：`store-description.md` の「短い説明」をコピー
- **詳細な説明**：`store-description.md` の「長い説明」をコピー
- **アイコン**：`icon-512.svg` を PNG 化（512×512）して提出
  - 注意：Play Store は PNG 必須。SVG → PNG 変換が必要
- **フィーチャーグラフィック**：1024×500 PNG（要作成）
- **スクリーンショット**：実機で 4〜8 枚撮影
  - オンボーディング画面
  - リーダー選択画面
  - 集中中（字が降ってる）
  - 休憩中（泡が上がってる）
  - 図鑑画面
  - 記録画面

### 3.3 コンテンツのレーティング
- アンケート回答 → ほぼ「該当なし」で **全年齢** になるはず

### 3.4 対象ユーザー
- **対象年齢**：13 歳以上 or 全年齢
- **ターゲット国／地域**：日本（最初は）

### 3.5 データセーフティ
`store-description.md` の「データセーフティ申告」を参照：
- **データ収集**：なし
- **データ共有**：なし
- **アカウント／プロフィール作成**：不要
- **データ暗号化**：該当なし（端末内のみ）

### 3.6 アプリへのアクセス
- ログイン情報不要：選択
- テスト手順：オンボーディングから順にプレイ可能、と記載

### 3.7 広告
- 含まれない：選択

## ✅ Step 4：AAB アップロード（内部テスト・15 分）

1. Play Console の左メニュー「**テスト → 内部テスト**」
2. 「新しいリリースを作成」
3. **AAB ファイルをアップロード**
4. リリース名：`v1.0.0`
5. リリースノート：
   ```
   初回リリース版。
   ・リーダー選択・字育成・特性スタッキング
   ・★1〜★6 段階解放
   ・バックグラウンド対応・引継ぎコード
   ```
6. 「保存 → 確認 → 内部テストとして公開」

### 4.1 テスター追加
1. 「テスター」タブ
2. メールリスト作成 → 自分のメアドを追加
3. 自分のスマホで Play ストアで「あなたのアプリ」と表示される
4. **インストールして実機で動作確認**

### 5.1 内部テストで 1〜3 日使い込む

### 5.2 製品版申請（クローズドテスト → オープンテスト → 製品版）
段階：
1. **クローズドテスト**（招待制 5〜10 人）
2. **オープンテスト**（公開ベータ）
3. **製品版**（一般公開）

各段階で **Google 審査が数日**。
製品版申請後、初回は **2 週間程度**かかることもある。
