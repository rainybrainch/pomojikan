ぽもじかんを「文字が降るタイマー」から **「文字育成ゲーム」** に昇華する。
ドラッグ&重ねる直感的メカニクスと段階的アンロックで、22,160 種類を活かす。

## 📋 8つの要件

### 1️⃣ バグ修正
- ボタンが機能していない箇所の特定・修正

### 2️⃣ 時間設定の拡張
- カスタム時間設定（1 分 1 秒単位で自由設定可能）
- 既存プリセット (25/5, 50/10, 15/3) は残す

### 3️⃣ パーティシステム
- ゲーム開始時に **4 種類のぽもじ** を選択
- その 4 種は **常に優先的に落ちてくる**
- 4 種の中から 1 体を「リーダー」と指定（これだけ消滅不可・成長可）

### 4️⃣ ぽもじの育成メカニクス
- **ドラッグ&ドロップで重ねる**：同じ種類のぽもじ同士を重ねると融合・進化
- **消滅システム**：不要なぽもじをタップして消滅 → 経験値に変換
- **リーダー + パーティ 3 種だけが Lv アップ**：4 体のみステータス保存・成長
  - 理由：メモリ・ストレージ効率化、パフォーマンス維持
  - その他 22,000 種は消滅経験値 or 図鑑コレクション
- **図鑑での成長確認**：4 体のレベル・進化状態が見える

### 5️⃣ レアリティ段階的解放
- 最初は **10 級クラス** のぽもじのみ落ちてくる
- **パーティ 4 体の平均 Lv** に応じて 9 級 → 8 級 →… → 伝説クラスと段階的に解放
- 級の設定・解放条件は PC 側で決定

### 6️⃣ バックグラウンド対応（改善版）
- アプリをバックグラウンドに送ると、タイマー一時停止
- フォアグラウンド復帰時、経過時間分のぽもじが **「いい感じに」落ちてくる**
  - 単なる一気落下ではなく、UX を考慮した演出（詳細は PC で相談）
  - 例：スローモーション、段階的落下、視覚効果など

### 7️⃣ UI・UX 改善
- 現在のメニューレイアウトを、ドラッグ&育成メインの画面に整理
- パーティ・リーダーの表示を明確に
- 4 体のステータス（Lv、経験値、進化状態）を常時見える位置に

### 8️⃣ アプリ申請
- **対象：Web 版（PWA）として今日中に完成・配布**
- ネイティブ化（iOS/Android）は後続判断

## 🏢 RBAI Phase 0 メモ（PC 引き継ぎ用）

### 重力／引力分類
**引力雨域（外向き・世間のため）** ・ ただし「ゲーム化」で重力比が上がる可能性（個人の美学が強く出る）。
分類：引力 70% / 重力 30%。

### 既存エコシステムへの影響
- **STATE.collection** に保存している 22,160 種類のうち、Lv 追跡対象は 4 体のみに変更
- **既存ユーザーのデータ保護**：v2.x までに集めた collection は読み取り専用で残す（移行モード）
- **shared_* キー**：影響なし（pomojikan_* / zaitoku_* prefix）
- **MAP.md** 接続線：UIKI 連携経由でユーザーアクション促進（パーティ未選択→「リーダーを選ぶ」タスク降雨等）

### 過去学習の適用
- v2.0 → v2.1 の反省「総数表記の誇大宣伝」 → **段階解放クラスの「級」表記は実数準拠**
- v2.5 の反省「データ追加で出現バランスが崩れる」 → **段階解放ロジックで露骨に解決**
- v2.6 categoryOf を **「級」判定にも応用可**

### 適用する成功パターン
- **シングル HTML + Vanilla JS + localStorage**（標準形式維持）
- **PWA + SW v29** （CACHE_NAME 更新で確実な配布）
- **核 1 空白の彫刻**：段階解放で「最初は 10 級だけ」と空白を保つ
- **核 2 数値は刻印**：4 体限定で Lv 数値の意味を保つ

### 独善チェック（暫定）
- ✅ 「パーティ 4 体」は遊びやすさと性能の両立、独善でない
- ⚠ 「リーダー 1 体は消滅不可」はゲームデザイン上の好み ← 引力作品としての一般受けは要検証
- ✅ 段階解放は「最初から全部見せない」= 核 1 空白の彫刻

## 🛠 技術設計の事前考察（PC 側 Phase 3 の叩き台）

### データ構造変更
```js
// v2.8 まで
STATE.collection = {
  '桜': { count: 12, firstSeen, lastSeen, rare, type, category }
  // 全 22,160 種類分が膨張
}

// v2.9 提案
STATE.collection = {
  // 既存（読み取り専用 = 図鑑用 archive）
  ...legacy
}
STATE.party = {
  hero: '桜',  // リーダー（消滅不可）
  members: ['月','風','雨'],  // パーティ 3 体
  // 各 Lv は collection.* に保存（既存構造を再利用）
}
STATE.classLevel = 10;  // 現在解放されている級（10→9→...→伝説）
```

### 級の定義（叩き台）
```
10 級：通常語のみ
 9 級：+ 一般名詞・動詞（POMOJI_NOUN/VERB）
 8 級：+ 形容詞・カタカナ
 7 級：+ 中学習漢字・季語
 6 級：+ 慣用句・色・食べ物
 5 級：+ オノマトペ・職業・道具
 4 級：+ 古語・人名用漢字・動植物
 3 級：+ 歴史人物・神話・宇宙
 2 級：+ 四字熟語・ことわざ・古典作品
 1 級：+ 故事成語・百人一首・専門用語
伝説：+ 創作・歌枕・漢詩・全カテゴリ
```
解放条件（叩き台）：パーティ 4 体の平均 Lv = (10 - 級) × 5

### ドラッグ&融合の物理
- 既存の `field.bodies[].x/y` を活用
- ポインターダウン → 該当 body の `dragging = true`
- ポインタームーブ → body 追従
- ドロップ位置で同種チェック → 融合 or 配置
- 既存 `mergeList` の進化ロジック（大→真→極）と併合可能

### 消滅システム
- タップ判定：シングルタップ + ホールド 0.5s で消滅
- 消滅 EXP = body.rare に応じて（rainbow=20 / legendary=10 / gold=5 / normal=1）
- ※ パーティ 4 体は消滅不可（dragging のみ可）

### バックグラウンド復帰
- 既存 `visibilitychange` で経過時間を計算
- 復帰時：経過時間分のスポーン数を計算 → **段階配信**
  - 例：30 体分 → 1 秒間隔で 5 体 × 6 波
  - 各波はスローモーション（vy *= 0.5）+ フェードイン

### サイズ判定
**XL**：複数機能・データ構造変更・UX 全面リニューアル
→ 全フェーズ実行（Phase 2.5 議論パート必須）
